get请求与post请求区别

  • POST 更加安全，不会作为 URL 的一部分，不会被缓存，保存在服务器日志、以及浏览器浏览记录中；
  • POST 发送的数据更大，GET 有 URL 长度限制；
  • POST 可以发送更多的数据类型，GET 只能发送 ASCII 字符；
  • POST 比 GET 慢；
  • POST 用数据的修改和写入，GET 一般用于搜索排序和筛选之类的操作

会话维持

运用背景：
在之前的请求之中，每次请求都相当于一个新的请求，也就是它们不是一个会话，不在同一个会话中就无法获得cookies，就像用一个浏览器逛淘宝一样，在不同的选项卡之间跳转，这样其实就是建立了一个长久会话。使用 post() 方法登录网页，然后再使用 get() 方法请求某个页面信息，如果不利用会话维持，将无法获取页面数据

维持方法：①两次请求设置一样的 cookies，缺点：繁琐；②使用 Session 对象
