基本介绍

selenium 是一个用于Web应用程序测试的工具。Selenium测试直接运行在浏览器中，就像真正的用户在操作一样。支持的浏览器包括IE（7, 8, 9, 10, 11），Mozilla Firefox，Safari，Google Chrome，Opera等。selenium 是一套完整的web应用程序测试系统,selenium可以模拟真实浏览器，自动化测试工具，支持多种浏览器，爬虫中主要用来解决JavaScript渲染问题。(本文所用浏览器为:chrome80.0.3987.132)

启动参数 作用
–user-agent=”” 设置请求头的 User-Agent
–window-size=xxx, xxx 设置浏览器分辨率

查找结点

Selenium 可以驱动浏览器完成各种操作，比如填充表单、模拟点击等。要完成这些操作，实现要知道在哪里点击，哪里填充，这就是 Selenium 节点查找

查找单个节点

查找情况: (如果某一种方法查找不到，就多换几种方法)

  • find_element_by_id 根据id查找
  • find_element_by_name 根据元素名查找
  • find_element_by_xpath 根据xpath查找
  • find_element_by_link_text 根据链接文本获取超链接
  • find_element_by_partial_link_text 根据部分链接文本查找
  • find_element_by_tag_name 根据标签名查找
  • find_element_by_class_name 根据class名查找
  • find_element_by_css_selector 根据css选择器查找

通用方法：find_element(),需要传递两个参数：查找方式 By 和值
例：driver.find_element_by_id(‘inner’) 等价于 find_element(By.ID, inner),使用时需要from selenium.webdriver.common.by import By

节点交互

Selenium 可以驱动浏览器来执行一些操作，也就是说可以让浏览器模拟执行一些动作。称为节点交互，比较常见的用法有：

  • send_keys：模拟按键输入
  • clear：清除元素的内容
  • click：单击元素
  • submit：提交表单

延时等待

现在不少web都都有使用ajax技术，即异步加载，有时候你要的东西都还没加载到你就去定位了，就会抛出异常。为了避免此等惨状，必须要待其加载一段时间后再进行后面的操作

隐式等待

当查找节点的时候，节点并没有立即出现，隐式等待将等待一段时间再查找该节点，使用 implicitly_wait()方法可以实现隐式等待,它的作用范围就是Webdriver对象实例的整个生命周期

显式等待

明确的要等待的元素在规定的时间之内都没找到,那么就抛出Exception

Code

在10秒内如果ID为kw的节点成功加载出来，就返回该节点；如果超过10秒还没有加载出来，就抛出异常。
基本方法：WebDriverWait(driver, 超时时长, 调用频率, 忽略异常).until(可执行方法, 超时时返回的信息)，超时时长默认为10s，调用频率默认为0.5s
