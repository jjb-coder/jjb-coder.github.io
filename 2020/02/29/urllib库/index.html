简介

python3的urllib包是一个url的处理包,内置在python3中,可以直接使用，用来模仿浏览器发送请求，包含了四个模块,request模块,parse(分析)模块，error模块,robotparser模块,以下针对这四个模块进行讲解,并结合一些实例进行练习,以下是对四个模块的基本简介:

模块 基本用途 常用程度
request 打开和读取url 常用
parse 解析url 常用
error 处理产生的异常错误 常用
robotparse 解析robot.txt文件 几乎不用

参数介绍

  • url:请求链接，必传参数
  • data:同urlopen
  • headers:指定发起http请求的头部信息,也可以调用Request实例的add_header()方法来添加请求头，添加请求头最常用的用法就是通过修改 User-Agent 来伪装浏览器，默认的 User-Agent是Python-urllib，我们可以通过修改它来伪装浏览器
  • origin_req_host：指的是请求方的 host名称或者IP地址,一般用不到
  • unverifiable 参数表示这个请求是否是无法验证的，默认值是False。意思就是说用户没有足够权限来选择接收这个请求的结果。例如我们请求一个HTML文档中的图片，但是我们没有自动抓取图像的权限，我们就要将 unverifiable 的值设置成 True，一般不用设置
  • method 参数指的是发起的 HTTP 请求的方式，有 GET、POST等

高级用法

常用函数介绍

  • ProxyHandler：为请求设置代理
  • HTTPCookieProcessor：处理 HTTP 请求中的 Cookies
  • HTTPDefaultErrorHandler：处理 HTTP 响应错误
  • HTTPPasswordMgr：用于管理密码，它维护了用户名密码的表
  • HTTPBasicAuthHandler：用于登录认证，一般和 HTTPPasswordMgr 结合使用
  • install_opener：用来创建（全局）默认opener
  • build_opener：用来创建opener对象

    Openers：
    当你获取一个URL,使用一个opener,正常情况下，我们使用默认opener,但是基本的urlopen()方法不支持代理、cookie等其他的HTTP/HTTPS高级功能。所以要支持这些功能：使用相关的Handler处理器来创建特定功能的处理器对象,然后通过urllib.request.build_opener()方法来使用这些处理器对象，创建自定义opener对象,使用自定义的opener对象，调用open()方法来发送请求求，如果程序里所有的请求都使用自定义的opener对象，可以使用urllib.request.install_opener()将自定义的opener对象定义为全局opener，表示如果之后凡是调用urlopen，都将使用这个opener(根据自己的需求来选择)

parse模块

常用函数

  • urlparse：实现url的识别和分段
  • urlunparse:实现url的构造
  • urljoin:将不完整链接拼成完整链接
  • urlencode:将字典形式的参数序列转化为url编码后的字符串（常用来构造POST和GET请求的参数）
  • quote:将中文转化为url编码格式(感觉挺重要的)

error模块

常用函数

  • URLError:打开一个不存在的页面，就会出现 URLError 错误，该错误有一个 reason 属性，用于返回错误的原因
  • HTTPError:URLError 的子类,专门用来处理 HTTP 请求错误

    HTTPError有三个属性：
    code:返回HTTP状态码，如404等
    reason：同父类一样，返回错误原因
    headers:返回请求头
